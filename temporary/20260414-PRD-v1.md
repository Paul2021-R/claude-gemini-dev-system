# PRD: AI Voice UI

## 1. 개요

Jarvis(nanobot) MCP 도구로 등록되는 음성 인터페이스.  
평소엔 백그라운드 프로세스로 대기, 핫키 감지 시 웹뷰가 열리고 음성 입출력을 처리.  
판단과 응답 생성은 전적으로 Jarvis에 위임 — 앱은 입출력 인터페이스만 담당.

---

## 2. 아키텍처

### 2-1. 전체 흐름

```
[백그라운드 상시 대기]
Python MCP 서버 + pynput 핫키 감지
      │
      │ (핫키 감지)
      ▼
[pywebview 윈도우 오픈] → LISTENING 상태 진입
      │
      │ (사용자 발화)
      ▼
[STT: Whisper.cpp] 음성 → 텍스트
      │
      │ (Discord Webhook POST → Jarvis 멘션 포함)
      ▼
[Discord 채널] `<@JARVIS_BOT_ID> {stt_text}`
      ▼
[Jarvis (nanobot)] allowFrom에 Webhook ID 등록 → 판단 → 응답 텍스트 생성
      │
      │ MCP function call: speak(text)
      ▼
[TTS: Supertone Stream] 텍스트 → 오디오 + 음소 타이밍
      │
      ▼
[React UI] SPEAKING + 단어 싱크 표시
      │
      ▼
[WAITING] → 다음 발화 대기 or 10s 후 윈도우 닫힘
```

### 2-2. 프로세스 구조

```
Python 단일 프로세스 (상시 실행)
├── MCP 서버 (Jarvis 도구 등록용)
│     └── tools: speak(text), cancel(), get_state()
├── pynput (글로벌 핫키 감지)
├── pywebview (핫키 시 윈도우 생성, 종료 시 destroy)
│     └── React + Three.js (Neural Sphere UI)
├── Whisper.cpp subprocess (STT)
└── Supertone API 클라이언트 (TTS)
```

### 2-3. Jarvis AGENTS.md 등록


## 3. 기술 스택

|구분|선택|비고|
|---|---|---|
|앱 컨테이너|pywebview (Cocoa/WebKit)|핫키 시 on-demand 윈도우|
|UI|React + TypeScript + Vite|Neural Sphere|
|3D 렌더링|Three.js (`@react-three/fiber`)|Sphere 애니메이션|
|MCP 서버|Python `mcp` 라이브러리|Jarvis 도구 노출|
|글로벌 핫키|`pynput`|상시 감지|
|TTS|Supertone Stream API (`include_phonemes=true`)|음소 타이밍 싱크|
|STT|Whisper.cpp (로컬)|오프라인, M4 Mac 최적화|
|오디오|Web Audio API `AnalyserNode`|amplitude 실시간 추출|
|JS↔Python|`window.pywebview.api`|브릿지, REST/SSE 불필요|

## 4. MCP 노출 툴 스펙

|툴|파라미터|동작|
|---|---|---|
|`speak`|`text: str`|TTS 변환 → SPEAKING 진입. 윈도우 없으면 자동 오픈|
|`cancel`|없음|재생 중단 → IDLE, 윈도우 닫기|
|`get_state`|없음|현재 상태 반환 (IDLE/LISTENING/SPEAKING/WAITING)|

---

## 5. Neural Sphere 상태

|상태|진입 조건|Sphere 행동|
|---|---|---|
|IDLE|기본 / 윈도우 닫힘 전|천천히 자전, dim|
|LISTENING|핫키 감지 후 윈도우 오픈|팽창 (~1.3x) + 마이크 amplitude 진동|
|PROCESSING|STT 완료 → Jarvis 응답 대기|빠른 pulse|
|SPEAKING|`speak()` 호출 시|TTS amplitude 연동 팽창/수축 + 단어 스트리밍|
|WAITING|재생 완료 후|토네이도 (나선형 회전), 10s 후 윈도우 닫힘|

- WAITING → 핫키 재입력: 바로 LISTENING
- WAITING → LISTENING 전환 시: scale burst 0.2s
- 색상: 노드 `#00e5ff` / 엣지 `rgba(0,229,255,0.2)` / 활성 glow

---

## 6. 텍스트 출력 (Phoneme-Synced)

Supertone `include_phonemes=true` 응답으로 음소 타이밍 → 단어 단위 스케줄링.

```
음소 start_times 기준 단어 그루핑
→ 오디오 재생 시작과 동시에 setTimeout 스케줄
→ 응답 전체 종료 후 텍스트 클리어
```

- fade-in / fade-out: 각 0.15s
- 폰트: weight 300 sans-serif, 흰색, Sphere 중앙

---

## 7. 윈도우 라이프사이클

```
프로세스 시작 → 윈도우 없음 (백그라운드 대기)
      │
      │ 핫키 or speak() 호출
      ▼
윈도우 생성 (pywebview)
      │
      │ WAITING 10s 경과 or cancel()
      ▼
윈도우 destroy (프로세스는 유지)
```

## 8. 환경변수 구성 (`.env`)

```env
# ─────────────────────────────────────
# Discord (STT → Jarvis 전달)
# ─────────────────────────────────────
DISCORD_WEBHOOK_URL=https://discord.com/api/webhooks/xxx/yyy
DISCORD_WEBHOOK_ID=                # Webhook ID (nanobot allowFrom 등록용)
DISCORD_JARVIS_BOT_ID=             # Jarvis 봇 ID (멘션용)

# ─────────────────────────────────────
# TTS (Supertone)
# ─────────────────────────────────────
SUPERTONE_API_KEY=
SUPERTONE_VOICE_ID=
SUPERTONE_STYLE=normal             # kind-default / normal / serene 등
SUPERTONE_LANGUAGE=ko              # ko / en / ja
SUPERTONE_MODEL=sona_speech_2      # sona_speech_1 / sona_speech_2

# ─────────────────────────────────────
# STT (Whisper.cpp)
# ─────────────────────────────────────
WHISPER_BIN_PATH=./bin/whisper
WHISPER_MODEL_PATH=./models/ggml-medium.bin  # 한국어 권장

# ─────────────────────────────────────
# 앱 일반
# ─────────────────────────────────────
HOTKEY=ctrl+space
WAITING_TIMEOUT_SEC=10
WINDOW_FULLSCREEN=true
WINDOW_FRAMELESS=true
```
